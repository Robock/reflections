What happens when you initialize a repository? Why do you need to do it?

When I initialize a repository, Git will start tracking the files and any changes that will be made in thefiles.


How is the staging area different from the working directory and the repository? What value do you think it offers?

The staging area is different from the working directory and the repository because it's an intermediatory stage for 
files to be added before they are commited.


How can you use the staging area to make sure you have one commit per logical change?

By adding one logical change and commiting just that the staging area helps to make sure one commit is made per
logical change.


What are some situations when branches would be helpful in keeping your history organized? How would branches help?

Branches would be helpful when you want your original file stay unchanged while you make some experiments or 
different versions.
